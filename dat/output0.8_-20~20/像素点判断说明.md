# 灰度图像像素点判断说明

## 数据来源

1. **原始数据文件**：
   - `LowEnergy/` 文件夹：每个位置的低能（GGAG）能量沉积数据
   - `HighEnergy/` 文件夹：每个位置的高能（GOS）能量沉积数据
   - 文件命名：`Y位置.txt`（例如：`0.0.txt`, `-20.0.txt`）
   - 每个文件包含128个像素的能量值（单位：keV），用制表符分隔

2. **数据读取**：
   ```python
   # 每个文件读取后得到128个能量值
   values = [float(x) for x in content.split('\t') if x.strip()]
   # 例如：[5694.22, 5017.53, 6340.86, ...] (128个值)
   ```

## 数据结构

### 数据数组形状
- `low_data`: 形状为 `(n_positions, 128)`
  - 第一维：扫描位置数量（例如：51个位置，从-20.0到20.0 mm）
  - 第二维：像素编号（0-127，共128个像素）

### 图像生成过程

1. **数据转置**：
   ```python
   low_gray = normalize_to_255(low_data.T)  # 转置后形状变为 (128, n_positions)
   ```
   - 转置前：`(51, 128)` - [位置, 像素]
   - 转置后：`(128, 51)` - [像素, 位置]

2. **归一化到0-255**：
   ```python
   def normalize_to_255(data):
       data_min = data.min()  # 找到最小值
       data_max = data.max()  # 找到最大值
       # 线性归一化公式
       normalized = ((data - data_min) / (data_max - data_min) * 255).astype(np.uint8)
       return normalized
   ```
   - 最小值 → 0（黑色）
   - 最大值 → 255（白色）
   - 中间值按比例线性映射

## 图像像素点对应关系

### 图像坐标系统
- **图像宽度（X轴）**：扫描位置数量（例如：51列）
  - 第0列对应位置 -20.0 mm
  - 第1列对应位置 -19.2 mm
  - ...
  - 第50列对应位置 20.0 mm

- **图像高度（Y轴）**：像素编号（例如：128行）
  - 第0行对应像素编号 0
  - 第1行对应像素编号 1
  - ...
  - 第127行对应像素编号 127

### 像素值计算示例

假设在位置 Y = 0.0 mm，像素编号 0 的能量沉积为 5694.22 keV：

1. **查找该位置在数组中的索引**：
   - 位置数组：`positions = [-20.0, -19.2, ..., 0.0, ..., 20.0]`
   - 位置 0.0 mm 的索引：假设为 `pos_idx = 25`

2. **获取能量值**：
   ```python
   energy_value = low_data[pos_idx, pixel_idx]  # low_data[25, 0] = 5694.22 keV
   ```

3. **归一化到0-255**：
   ```python
   # 假设整个数据集的最小值为 3000.0，最大值为 8000.0
   normalized = (5694.22 - 3000.0) / (8000.0 - 3000.0) * 255
   normalized = 2694.22 / 5000.0 * 255 ≈ 137
   ```

4. **图像中的位置**：
   - 图像坐标：`(列=25, 行=0)`
   - 像素值：`137`（灰度值，0-255范围）

```272:285:code/build/combine_and_visualize.py
def normalize_to_255(data):
    """
    将数据归一化到0-255范围
    """
    data_min = data.min()
    data_max = data.max()
    
    if data_max == data_min:
        # 如果所有值都相同，返回全零或全255
        return np.zeros_like(data, dtype=np.uint8)
    
    # 线性归一化到0-255
    normalized = ((data - data_min) / (data_max - data_min) * 255).astype(np.uint8)
    return normalized
```

## 总结

**图像像素点的判断流程**：
1. 从文本文件读取每个位置的128个像素能量值
2. 将所有位置的数据组合成 `(n_positions, 128)` 数组
3. 转置数组得到 `(128, n_positions)`，匹配图像显示方向
4. 对每个能量值进行线性归一化：`(值 - 最小值) / (最大值 - 最小值) * 255`
5. 将归一化后的值（0-255）作为灰度图像的像素值
6. 图像中：**行 = 像素编号（0-127），列 = 扫描位置索引（0到n_positions-1）**

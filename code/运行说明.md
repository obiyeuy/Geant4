# CZT 探测器模拟项目运行说明

## 一、项目构建（首次运行或修改代码后）

### 方法1：使用 CMake 构建

```bash
# 进入项目目录
cd /home/yyb/workspace/XRay-all/XRay-detectionCode/code

# 创建并进入构建目录
mkdir -p build
cd build

# 运行 CMake 配置
cmake ..

# 编译项目
make

# 编译完成后，可执行文件 CZT 会生成在 build 目录中
```

### 方法2：如果已经构建过（直接运行）

如果 build 目录中已经有 `CZT` 可执行文件，可以跳过构建步骤。

---

## 二、运行模拟

### 方法1：批处理模式运行（推荐，用于产生输出文件）

```bash
# 进入 build 目录
cd /home/yyb/workspace/XRay-all/XRay-detectionCode/code/build

# 运行模拟（使用 run.mac 脚本）
./CZT run.mac
```

**说明：**
- `run.mac` 是运行脚本，包含模拟参数
- 当前设置：运行 1000 个事件（可在 run.mac 中修改）
- 运行完成后会在 `build` 目录生成输出文件

### 方法2：交互式模式运行（用于调试和可视化）

```bash
# 进入 build 目录
cd /home/yyb/workspace/XRay-all/XRay-detectionCode/code/build

# 启动交互式模式
./CZT

# 在交互式命令行中可以执行：
# /control/execute run.mac    # 执行运行脚本
# /control/execute vis.mac    # 启动可视化
# /run/beamOn 1000            # 直接运行事件
```

## 三、输出文件位置和内容

运行完成后，输出文件会生成在 **build 目录**中：

### 1. `Mydata.root` - ROOT 格式数据文件
- **位置**：`build/Mydata.root`
- **内容**：256 个直方图（每个像素的能量沉积分布）
  - Pixel NO. 1000-1127：GGAG 材料（128个像素）
  - Pixel NO. 2000-2127：GOS 材料（128个像素）
- **查看方式**：
  ```bash
  # 使用 ROOT 查看
  root Mydata.root
  # 在 ROOT 中：
  # TBrowser b;  # 打开浏览器查看直方图
  ```

### 2. `ListMode1.txt` - GGAG 材料总能量沉积
- **位置**：`build/ListMode1.txt`
- **内容**：128 个数值（用制表符分隔），每个像素的累积能量沉积
- **查看方式**：
  ```bash
  cat ListMode1.txt
  # 或
  less ListMode1.txt
  ```

### 3. `ListMode2.txt` - GOS 材料总能量沉积
- **位置**：`build/ListMode2.txt`
- **内容**：128 个数值（用制表符分隔），每个像素的累积能量沉积
- **查看方式**：
  ```bash
  cat ListMode2.txt
  # 或
  less ListMode2.txt
  ```

## 四、修改运行参数

### 修改事件数量

编辑 `run.mac` 文件（或 `build/run.mac`）：

```bash
# 修改这一行来改变事件数量
/run/beamOn 1000000    # 运行 100 万个事件（更精确但更慢）
/run/beamOn 1000       # 运行 1000 个事件（快速测试）
```

### 修改粒子源参数

编辑 `src/PrimaryGeneratorAction.cc`：
- 粒子能量：第 29 行 `SetParticleEnergy(160.0 * keV)`
- 发射位置：第 40 行 `SetParticlePosition(...)`
- 角度范围：第 43-46 行

修改后需要重新编译：
```bash
cd build
make
```

## 六、注意事项

1. **运行时间**：事件数量越多，运行时间越长
   - 1000 个事件：通常几秒到几分钟
   - 1000000 个事件：可能需要几小时

2. **输出文件位置**：所有输出文件都在 `build` 目录中

3. **ROOT 文件**：需要安装 ROOT 软件才能查看 `.root` 文件

4. **重新运行**：每次运行会**覆盖**之前的输出文件

5. **并行运行**：如果需要多次运行，建议：
   - 每次运行前备份输出文件
   - 或修改代码将输出文件命名包含时间戳

---

## 七、验证运行是否成功

运行成功后，你应该看到：
- 终端显示事件进度（每 1000 个事件打印一次）
- 最后显示 "Using ROOT" 或类似信息
- `build` 目录中出现三个输出文件

如果出现错误，检查：
- Geant4 是否正确安装
- 是否有写入权限
- 磁盘空间是否充足

# 矿石分选式步进扫描实现方案

## 一、当前系统理解

### 1.1 物体位置控制机制

从代码分析可以看出：

- **控制命令**：`/Xray/det/SetObjShift <value> mm`
- **作用**：控制待测物体在 **Y 方向**的位置偏移（第249行：`G4ThreeVector(0,ObjShift,0)`）
- **当前实现**：物体是一个球体（`sphereRadius = 15 mm`），位于 `(0, ObjShift, 0)` 位置
- **材料**：磷酸钙（CalciumPhosphate）

### 1.2 输出文件机制

每次运行会产生三个固定名称的输出文件：
- `Mydata.root` - ROOT 格式，包含 256 个直方图（每个像素的能量沉积分布）
- `ListMode1.txt` - GGAG 材料的总能量沉积（128个像素，制表符分隔）
- `ListMode2.txt` - GOS 材料的总能量沉积（128个像素，制表符分隔）

**问题**：当前实现中，每次运行都会**覆盖**之前的输出文件。

---

## 二、步进扫描需求分析

### 2.1 扫描场景

类似矿石分选，需要：
1. **步进移动**：物体在 Y 方向按一定步长移动（例如：-20mm 到 +20mm，步长 1mm）
2. **多次运行**：每个位置运行一次模拟
3. **数据组合**：将所有位置的输出数据组合起来，形成：
   - **空间-能量分布图**：不同 Y 位置对应的能量沉积
   - **像素响应图**：每个像素在不同 Y 位置的响应
   - **总能量分布**：不同位置的总能量沉积

### 2.2 数据组合目标

最终需要得到：
- **3D 数据**：`[Y位置, 像素编号, 能量沉积]`
- **2D 投影**：`[Y位置, 像素编号]` 对应的总能量
- **1D 曲线**：每个像素随 Y 位置变化的能量响应曲线

## 三、实现方案对比

### 方案A：外部脚本循环（推荐，无需修改代码）

#### 优点：
- ✅ **无需修改 C++ 代码**
- ✅ **灵活**：可以随时调整扫描参数
- ✅ **易于调试**：每次运行独立，可以单独检查
- ✅ **并行化友好**：可以并行运行多个位置

#### 实现思路：
1. 创建 Shell 脚本或 Python 脚本
2. 循环改变 `SetObjShift` 值
3. 每次运行前修改 `run.mac` 或动态生成宏文件
4. 每次运行后重命名输出文件（添加位置标识）
5. 最后用脚本组合所有数据

#### 工作流程：
```
for Y_position in [-20, -19, ..., 0, ..., 19, 20] mm:
    1. 生成 run_scan_Y{position}.mac 文件
    2. 运行: ./CZT run_scan_Y{position}.mac
    3. 重命名输出:
       - Mydata.root → Mydata_Y{position}.root
       - ListMode1.txt → ListMode1_Y{position}.txt
       - ListMode2.txt → ListMode2_Y{position}.txt
    4. 继续下一个位置

最后：
    5. 用 Python/ROOT 脚本读取所有文件
    6. 组合成 3D 数组
    7. 生成分析结果
```

---

### 方案B：修改代码支持多位置运行（需要修改代码）

#### 优点：
- ✅ **自动化**：一次运行完成所有位置
- ✅ **数据格式统一**：可以直接输出组合后的数据

#### 缺点：
- ❌ **需要修改 C++ 代码**
- ❌ **灵活性差**：修改参数需要重新编译
- ❌ **调试困难**：单个位置出错需要重新运行全部

#### 实现思路：
1. 在 `RunAction` 中添加扫描参数（起始位置、结束位置、步长）
2. 在 `BeginOfRunAction` 中循环不同位置
3. 每次位置改变后：
   - 调用 `DetectorConstruction::SetObjShiftDistance()`
   - 调用 `G4RunManager::ReinitializeGeometry()`
   - 运行事件
4. 输出文件包含位置信息（例如：在文件名中添加位置索引）

---

### 方案C：混合方案（推荐用于生产环境）

#### 思路：
- **外部脚本**：控制扫描流程和文件管理
- **代码修改**：让输出文件名包含位置信息（小改动）

#### 优点：
- ✅ 结合两种方案的优点
- ✅ 代码改动最小（只改输出文件名）
- ✅ 外部脚本灵活控制

---

## 四、推荐方案详细设计（方案A + 部分C）

### 4.1 文件命名策略

**输出文件命名规则**：
- `Mydata_Y{position}.root` - 例如：`Mydata_Y-20.0.root`, `Mydata_Y0.0.root`
- `ListMode1_Y{position}.txt` - 例如：`ListMode1_Y-20.0.txt`
- `ListMode2_Y{position}.txt` - 例如：`ListMode2_Y-20.0.txt`

**或者使用索引**：
- `Mydata_scan{index:03d}.root` - 例如：`Mydata_scan000.root`, `Mydata_scan001.root`
- 同时保存位置映射文件：`scan_positions.txt`（记录索引对应的 Y 位置）

### 4.2 扫描脚本结构

#### Shell 脚本示例逻辑：
```bash
#!/bin/bash

# 扫描参数
START_Y=-20.0    # 起始位置 (mm)
END_Y=20.0       # 结束位置 (mm)
STEP=1.0         # 步长 (mm)
EVENTS=1000      # 每个位置的事件数

# 创建输出目录
mkdir -p scan_results

# 循环扫描
for Y in $(seq $START_Y $STEP $END_Y); do
    echo "Scanning at Y = $Y mm"
    
    # 生成临时宏文件
    cat > scan_temp.mac << EOF
/Xray/det/SetObjShift $Y mm
/run/initialize
/control/verbose 1
/run/verbose 1
/run/beamOn $EVENTS
EOF
    
    # 运行模拟
    ./CZT scan_temp.mac
    
    # 重命名并移动输出文件
    mv Mydata.root scan_results/Mydata_Y${Y}.root
    mv ListMode1.txt scan_results/ListMode1_Y${Y}.txt
    mv ListMode2.txt scan_results/ListMode2_Y${Y}.txt
    
    echo "Completed Y = $Y mm"
done

# 清理临时文件
rm scan_temp.mac

echo "Scan completed! Results in scan_results/"
```

### 4.3 数据组合脚本（Python 示例）

```python
import numpy as np
import uproot  # 用于读取 ROOT 文件
import glob
import os

# 1. 读取所有位置的文本文件
def load_text_data(result_dir):
    positions = []
    listmode1_data = []  # GGAG
    listmode2_data = []  # GOS
    
    # 找到所有 ListMode1 文件
    files = sorted(glob.glob(f"{result_dir}/ListMode1_Y*.txt"))
    
    for file in files:
        # 从文件名提取位置
        y_pos = float(file.split("_Y")[1].split(".txt")[0])
        positions.append(y_pos)
        
        # 读取数据（制表符分隔）
        data1 = np.loadtxt(file)
        data2 = np.loadtxt(file.replace("ListMode1", "ListMode2"))
        
        listmode1_data.append(data1)
        listmode2_data.append(data2)
    
    return np.array(positions), np.array(listmode1_data), np.array(listmode2_data)

# 2. 读取 ROOT 文件（如果需要直方图数据）
def load_root_histograms(result_dir, pixel_id=1000):
    positions = []
    histograms = []
    
    files = sorted(glob.glob(f"{result_dir}/Mydata_Y*.root"))
    
    for file in files:
        y_pos = float(file.split("_Y")[1].split(".root")[0])
        positions.append(y_pos)
        
        # 使用 uproot 读取 ROOT 文件
        with uproot.open(file) as f:
            hist_name = f"Pixel NO. {pixel_id}"
            hist = f[hist_name].to_numpy()
            histograms.append(hist)
    
    return np.array(positions), np.array(histograms)

# 3. 生成组合结果
def generate_combined_results(result_dir):
    positions, ggag_data, gos_data = load_text_data(result_dir)
    
    # 创建 3D 数据： [Y位置索引, 像素编号, 能量]
    # ggag_data 形状: [n_positions, 128_pixels]
    
    # 保存组合数据
    np.savez(f"{result_dir}/combined_data.npz",
             positions=positions,
             ggag_energy=ggag_data,
             gos_energy=gos_data)
    
    # 生成 2D 热图：位置 vs 像素
    import matplotlib.pyplot as plt
    
    fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(14, 6))
    
    # GGAG 热图
    im1 = ax1.imshow(ggag_data.T, aspect='auto', origin='lower',
                     extent=[positions.min(), positions.max(), 0, 127])
    ax1.set_xlabel('Y Position (mm)')
    ax1.set_ylabel('Pixel Number')
    ax1.set_title('GGAG Energy Deposition')
    plt.colorbar(im1, ax=ax1, label='Energy (keV)')
    
    # GOS 热图
    im2 = ax2.imshow(gos_data.T, aspect='auto', origin='lower',
                     extent=[positions.min(), positions.max(), 0, 127])
    ax2.set_xlabel('Y Position (mm)')
    ax2.set_ylabel('Pixel Number')
    ax2.set_title('GOS Energy Deposition')
    plt.colorbar(im2, ax=ax2, label='Energy (keV)')
    
    plt.tight_layout()
    plt.savefig(f"{result_dir}/scan_heatmap.png", dpi=300)
    print(f"Saved heatmap to {result_dir}/scan_heatmap.png")
    
    return positions, ggag_data, gos_data
```

## 五、代码修改建议（最小改动）

如果要采用方案C（混合方案），只需要**小幅度修改** `RunAction.cc`：

### 5.1 修改输出文件名（添加位置参数）

**思路**：在 `RunAction` 中添加一个成员变量来存储当前扫描位置，输出文件名包含位置信息。

**需要修改的地方**：
1. `RunAction.hh`：添加 `G4double fScanPosition;` 成员变量和设置函数
2. `RunAction.cc`：在 `BeginOfRunAction` 中使用位置信息生成文件名
3. 或者：通过环境变量传递位置信息（更简单，无需修改头文件）

**环境变量方案（最简单）**：
```cpp
// 在 RunAction.cc 的 BeginOfRunAction 中
G4String scanPos = getenv("SCAN_POSITION") ? getenv("SCAN_POSITION") : "0.0";
G4String fileName = "Mydata_Y" + scanPos + ".root";
```

这样外部脚本只需要：
```bash
export SCAN_POSITION="-20.0"
./CZT run.mac
```

## 六、完整工作流程示例

### 6.1 扫描执行流程

```
1. 准备阶段
   ├─ 确定扫描范围：Y = -20mm 到 +20mm
   ├─ 确定步长：1mm
   └─ 确定每个位置的事件数：1000

2. 扫描循环（41个位置）
   ├─ Y = -20.0 mm → 运行 → 保存结果
   ├─ Y = -19.0 mm → 运行 → 保存结果
   ├─ ...
   ├─ Y = 0.0 mm → 运行 → 保存结果
   ├─ ...
   └─ Y = 20.0 mm → 运行 → 保存结果

3. 数据组合
   ├─ 读取所有位置的文本文件
   ├─ 读取所有位置的 ROOT 文件（可选）
   ├─ 组合成 3D 数组
   └─ 生成分析图表

4. 结果输出
   ├─ combined_data.npz（NumPy 格式）
   ├─ scan_heatmap.png（2D 热图）
   ├─ pixel_response_curves.png（像素响应曲线）
   └─ scan_summary.txt（统计摘要）
```

### 6.2 数据分析目标

组合后的数据可以用于：
1. **空间分辨率分析**：不同 Y 位置的能量分布
2. **像素响应特性**：每个像素的响应曲线
3. **材料识别**：GGAG 和 GOS 的响应差异
4. **最优位置确定**：找到能量沉积最大的位置
5. **分选算法开发**：基于空间-能量分布的分选逻辑

---

## 七、注意事项

### 7.1 性能考虑

- **运行时间**：41个位置 × 1000事件 ≈ 41倍单次运行时间
- **存储空间**：41个 ROOT 文件可能较大（取决于事件数和直方图精度）
- **并行化**：可以同时运行多个位置（如果有多核/多机器）

### 7.2 数据一致性

- **随机种子**：如果需要可重复性，需要控制随机种子
- **事件数**：每个位置的事件数应该相同，便于比较
- **文件命名**：确保文件名唯一且可解析

### 7.3 错误处理

- **运行失败**：某个位置运行失败时，应该记录并继续
- **文件检查**：运行后检查输出文件是否存在且非空
- **数据验证**：组合数据前验证所有文件完整性

---

## 八、总结

**推荐方案**：**方案A（外部脚本）+ 部分方案C（环境变量传递位置）**

**理由**：
1. ✅ **代码改动最小**：只需在输出文件名中使用环境变量
2. ✅ **灵活性高**：扫描参数完全由脚本控制
3. ✅ **易于扩展**：可以轻松添加并行化、断点续传等功能
4. ✅ **调试友好**：可以单独运行某个位置进行测试

**实施步骤**：
1. 修改 `RunAction.cc` 使用环境变量生成输出文件名（可选，但推荐）
2. 创建扫描脚本（Shell 或 Python）
3. 创建数据组合脚本（Python）
4. 运行扫描
5. 分析结果

这样既保持了代码的简洁性，又实现了灵活的数据组合分析功能。

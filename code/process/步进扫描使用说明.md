# 步进扫描使用说明

## 概述

本方案使用外部脚本循环的方式实现步进扫描，无需修改C++代码。包含两个主要脚本：
1. `scan_step.sh` - 步进扫描脚本
2. `combine_and_visualize.py` - 数据组合和可视化脚本

## 文件结构

```
code/
├── scan_step.sh              # 步进扫描脚本
├── combine_and_visualize.py  # 数据组合和可视化脚本
├── build/
│   ├── CZT                   # 可执行文件
│   └── output/               # 输出目录
│       ├── LowEnergy/        # 低能数据（GGAG）
│       ├── HighEnergy/       # 高能数据（GOS）
│       └── Mydata/           # ROOT文件
```

## 使用步骤

### 1. 准备工作

确保项目已编译：
```bash
cd /home/yyb/workspace/XRay-all/XRay-detectionCode/code/build
make
```

### 2. 运行步进扫描

编辑 `scan_step.sh` 中的扫描参数（如需要）：
```bash
START_Y=-20.0      # 起始位置 (mm)
END_Y=20.0         # 结束位置 (mm)
STEP=1.0           # 步长 (mm)
EVENTS=10000       # 每个位置的事件数
```

运行扫描脚本：
```bash
cd /home/yyb/workspace/XRay-all/XRay-detectionCode/code
bash scan_step.sh
```

或者：
```bash
cd /home/yyb/workspace/XRay-all/XRay-detectionCode/code
./scan_step.sh
```

### 3. 数据组合和可视化

扫描完成后，运行可视化脚本：
```bash
cd /home/yyb/workspace/XRay-all/XRay-detectionCode/code
python3 combine_and_visualize.py
```

**注意**：需要安装以下Python库：
```bash
pip install numpy matplotlib
```

## 输出结果

### 扫描输出

扫描完成后，在 `build/output/` 目录下会生成：
- `LowEnergy/{位置}.txt` - 每个位置的低能数据（128个像素）
- `HighEnergy/{位置}.txt` - 每个位置的高能数据（128个像素）
- `Mydata/{位置}.root` - 每个位置的ROOT文件

例如：
```
output/
├── LowEnergy/
│   ├── -20.0.txt
│   ├── -19.0.txt
│   ├── ...
│   └── 20.0.txt
├── HighEnergy/
│   ├── -20.0.txt
│   ├── -19.0.txt
│   ├── ...
│   └── 20.0.txt
└── Mydata/
    ├── -20.0.root
    ├── -19.0.root
    ├── ...
    └── 20.0.root
```

### 可视化输出

运行 `combine_and_visualize.py` 后，会在 `build/output/` 目录下生成：

1. **scan_heatmap.png** - 2D热图
   - 显示每个像素在不同Y位置的能量沉积
   - 左图：低能（GGAG）
   - 右图：高能（GOS）

2. **pixel_response_curves.png** - 像素响应曲线
   - 显示几个代表性像素随位置变化的能量响应
   - 可以观察像素的空间响应特性

3. **total_energy_vs_position.png** - 总能量曲线
   - 显示低能、高能和总能量随位置的变化
   - 用于分析最优检测位置

4. **position_slices.png** - 位置切片图
   - 显示几个代表性位置的像素能量分布
   - 可以观察不同位置的能量分布模式

5. **combined_data.npz** - 组合数据（NumPy格式）
   - 包含所有位置的完整数据
   - 可用于进一步分析

6. **combined_data_summary.txt** - 数据摘要
   - 每个位置的总能量统计

## 自定义参数

### 修改扫描范围

编辑 `scan_step.sh`：
```bash
START_Y=-30.0      # 修改起始位置
END_Y=30.0         # 修改结束位置
STEP=0.5           # 修改步长（更精细的扫描）
EVENTS=50000       # 修改事件数（更精确但更慢）
```

### 修改可视化参数

编辑 `combine_and_visualize.py` 中的参数：
```python
NUM_PIXELS = 128  # 像素数量（通常不需要修改）

# 在函数中可以修改：
num_pixels_to_show = 10  # 显示更多像素的响应曲线
num_slices = 7           # 显示更多位置切片
```

## 性能考虑

- **扫描时间**：总时间 = 位置数 × 每个位置的运行时间
  - 例如：41个位置 × 每个位置1分钟 = 约41分钟
  
- **存储空间**：每个位置生成3个文件
  - 文本文件：约几KB
  - ROOT文件：取决于事件数和直方图精度

- **并行化**：可以手动并行运行多个位置
  ```bash
  # 在终端1运行位置 -20 到 -10
  # 在终端2运行位置 -10 到 0
  # 在终端3运行位置 0 到 10
  # 在终端4运行位置 10 到 20
  ```

## 故障排除

### 问题1：找不到可执行文件
**解决**：确保在 `build` 目录中已编译项目
```bash
cd build && make
```

### 问题2：Python库缺失
**解决**：安装所需库
```bash
pip install numpy matplotlib
```

### 问题3：数据文件不完整
**解决**：
- 检查每个位置的运行是否成功
- 检查磁盘空间是否充足
- 可以单独运行某个位置进行测试

### 问题4：可视化图像显示异常
**解决**：
- 检查数据文件是否完整
- 确认所有位置的数据都已生成
- 检查Python版本（建议Python 3.6+）

## 进阶使用

### 只扫描特定位置

修改 `scan_step.sh`，使用数组：
```bash
# 只扫描特定位置
for Y in -10.0 -5.0 0.0 5.0 10.0; do
    # ... 扫描代码 ...
done
```

### 断点续传

如果扫描中断，可以：
1. 检查已生成的文件
2. 修改 `scan_step.sh` 跳过已存在的位置
3. 或者手动删除已生成的文件后重新运行

### 自定义可视化

可以修改 `combine_and_visualize.py` 添加自定义分析：
- 添加新的可视化函数
- 修改图像样式和颜色
- 添加统计分析
